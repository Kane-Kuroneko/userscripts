# CI/CD 流程图示

> 可视化展示新旧 CI/CD 流程对比

---

## 📊 旧方案流程 (v1)

```
┌─────────────────────────────────────────────────────────┐
│                    开发者本地环境                          │
│                                                         │
│  1. 编写代码                                              │
│     ↓                                                   │
│  2. git commit                                          │
│     ↓                                                   │
│  3. git push                                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────┐
│                   GitHub 仓库                            │
│                                                         │
│  4. 打开 GitHub Releases 页面 (手动)                     │
│     ↓                                                   │
│  5. 点击 "Create a new release" (手动)                  │
│     ↓                                                   │
│  6. 填写版本号、标题、描述 (手动)                         │
│     ↓                                                   │
│  7. 点击 "Publish release" (手动)                       │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 触发
┌─────────────────────────────────────────────────────────┐
│              GitHub Actions (main.yml)                   │
│                                                         │
│  8. Checkout 代码                                       │
│     ↓                                                   │
│  9. 安装依赖 (固定: switch520-auto-secret)               │
│     ↓                                                   │
│  10. 构建项目 (固定路径)                                 │
│     ↓                                                   │
│  11. 上传产物到已存在的 Release                          │
│     ↓                                                   │
│  12. 触发 GreasyFork Webhook                            │
└─────────────────────────────────────────────────────────┘

问题:
❌ 步骤 4-7 需要手动操作
❌ 只能构建 switch520-auto-secret
❌ 添加新项目需修改 CI 配置
❌ 容易填错版本信息
```

## ✨ 新方案流程 (v2)

```
┌─────────────────────────────────────────────────────────┐
│                    开发者本地环境                          │
│                                                         │
│  1. 编写代码                                              │
│     ↓                                                   │
│  2. git commit                                          │
│     ↓                                                   │
│  3. git push                                            │
│     ↓                                                   │
│  4. git tag switch520-auto-secret/v1.5.0               │
│     ↓                                                   │
│  5. git push origin tag                                 │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 自动触发
┌─────────────────────────────────────────────────────────┐
│           GitHub Actions (release.yml)                   │
│                                                         │
│  6. 解析 Tag → 项目名 + 版本号                           │
│     switch520-auto-secret/v1.5.0                        │
│     ├─ 项目: switch520-auto-secret                      │
│     └─ 版本: v1.5.0                                    │
│     ↓                                                   │
│  7. 验证项目是否存在                                     │
│     ↓                                                   │
│  8. 安装依赖 (动态: $PROJECT_NAME)                       │
│     ↓                                                   │
│  9. 构建项目 (动态: npm run build:$PROJECT_NAME)        │
│     ↓                                                   │
│  10. 验证构建产物                                       │
│     ↓                                                   │
│  11. 自动创建 Release                                   │
│     ├─ Tag: switch520-auto-secret/v1.5.0               │
│     ├─ 名称: switch520-auto-secret v1.5.0              │
│     └─ 附件: dist/**                                   │
│     ↓                                                   │
│  12. 触发 GreasyFork Webhook                            │
└─────────────────────────────────────────────────────────┘
                     │
                     ↓ 开发者后续操作
┌─────────────────────────────────────────────────────────┐
│                  补充 Release Notes                       │
│                                                         │
│  13. 打开 Release 页面                                  │
│     ↓                                                   │
│  14. 点击编辑按钮                                       │
│     ↓                                                   │
│  15. 填写详细更新日志                                    │
│     ↓                                                   │
│  16. 保存 (触发 patch notes 同步)                        │
└─────────────────────────────────────────────────────────┘

优势:
✅ 全自动化，无需手动创建 Release
✅ 支持任意项目，零配置
✅ 版本信息从 Tag 自动解析
✅ 减少人为错误
```

## 🔄 多项目发布流程

### 场景: 同时发布两个项目

```
开发者执行:
─────────────────────────────────────────────────────────
git tag switch520-auto-secret/v1.5.0
git tag github-enhancer/v0.2.0
git push origin switch520-auto-secret/v1.5.0 github-enhancer/v0.2.0
─────────────────────────────────────────────────────────
                     │
        ┌────────────┴────────────┐
        ↓                         ↓
┌───────────────────┐    ┌───────────────────┐
│  CI 触发 #1       │    │  CI 触发 #2       │
│                   │    │                   │
│  解析:            │    │  解析:            │
│  switch520-...    │    │  github-enhancer  │
│                   │    │                   │
│  构建:            │    │  构建:            │
│  switch520-...    │    │  github-enhancer  │
│                   │    │                   │
│  创建 Release #1  │    │  创建 Release #2  │
└───────────────────┘    └───────────────────┘

结果:
✅ 两个项目独立构建
✅ 两个 Release 独立创建
✅ 互不干扰
```

## 📦 Patch Notes 同步流程

```
开发者编辑 Release (补充更新日志)
         │
         ↓ 触发
┌─────────────────────────────────────────────────────────┐
│       GitHub Actions (sync-patch-notes.yml)              │
│                                                         │
│  1. 解析 Release Tag 获取项目名                          │
│     ↓                                                   │
│  2. Checkout patch-notes 分支                           │
│     ↓                                                   │
│  3. 生成项目特定的 patch notes                           │
│     patch-notes-switch520-auto-secret.html              │
│     ↓                                                   │
│  4. 提交并推送                                          │
└─────────────────────────────────────────────────────────┘

结果:
patch-notes 分支/
├── patch-notes-switch520-auto-secret.html  ← 项目 A
├── patch-notes-github-enhancer.html        ← 项目 B
└── patch-notes-test-tampermonkey-jsx.html  ← 项目 C

优势:
✅ 各项目独立，不会覆盖
✅ 便于追溯历史
✅ 支持单独更新
```

## 🛠️ 使用 release.sh 脚本流程

```
┌─────────────────────────────────────────────────────────┐
│              运行发布脚本                                  │
│                                                         │
│  $ ./scripts/release.sh switch520-auto-secret 1.5.0    │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 自动执行
┌─────────────────────────────────────────────────────────┐
│                  脚本内部流程                              │
│                                                         │
│  1. 参数验证                                             │
│     ├─ 项目名: switch520-auto-secret ✅                 │
│     └─ 版本号: 1.5.0 ✅                                 │
│     ↓                                                   │
│  2. 检查项目存在性                                       │
│     ├─ projects/switch520-auto-secret ✅                │
│     ↓                                                   │
│  3. 检查 Tag 冲突                                       │
│     ├─ switch520-auto-secret/v1.5.0 不存在 ✅           │
│     ↓                                                   │
│  4. 确认发布 (y/N)                                      │
│     └─ 用户输入: y                                      │
│     ↓                                                   │
│  5. 本地构建测试                                         │
│     ├─ npm run build:switch520-auto-secret             │
│     ├─ 构建成功 ✅                                      │
│     └─ 验证 dist/ 目录 ✅                               │
│     ↓                                                   │
│  6. 检查未提交更改                                       │
│     └─ 无更改 / 用户确认继续 ✅                          │
│     ↓                                                   │
│  7. 创建 Tag                                            │
│     ├─ git tag -a switch520-auto-secret/v1.5.0         │
│     └─ 成功 ✅                                          │
│     ↓                                                   │
│  8. 推送 Tag                                            │
│     ├─ git push origin switch520-auto-secret/v1.5.0   │
│     └─ 成功 ✅                                          │
│     ↓                                                   │
│  9. 显示提示信息                                         │
│     ├─ CI/CD 进度链接                                   │
│     ├─ Release 链接                                     │
│     └─ 提醒补充 Release Notes                           │
└─────────────────────────────────────────────────────────┘
                     │
                     ↓ 自动触发 CI/CD
┌─────────────────────────────────────────────────────────┐
│              GitHub Actions 接管                          │
│          (同新方案流程 6-12 步)                           │
└─────────────────────────────────────────────────────────┘
```

## 🎯 错误处理流程

### 场景 1: 构建失败

```
推送 Tag
   ↓
CI 触发
   ↓
构建失败 ❌
   ↓
┌─────────────────────────────────────────────────────────┐
│                  错误处理                                 │
│                                                         │
│  1. CI 标记为失败                                       │
│     ↓                                                   │
│  2. 发送通知 (邮件/GitHub)                              │
│     ↓                                                   │
│  3. Release 不会创建                                    │
│     ↓                                                   │
│  4. 开发者收到通知                                      │
│     ↓                                                   │
│  5. 查看 CI 日志定位问题                                │
│     ↓                                                   │
│  6. 修复代码                                            │
│     ↓                                                   │
│  7. 删除失败 Tag                                        │
│     git push --delete origin project/v1.0.0            │
│     ↓                                                   │
│  8. 重新推送新版本                                      │
│     git tag project/v1.0.1                             │
│     git push origin project/v1.0.1                     │
└─────────────────────────────────────────────────────────┘
```

### 场景 2: Tag 已存在

```
$ git tag switch520-auto-secret/v1.5.0
         ↓
    Tag 已存在 ❌
         ↓
┌─────────────────────────────────────────────────────────┐
│  release.sh 脚本会:                                      │
│                                                         │
│  ❌ 报错: "Tag 'switch520-auto-secret/v1.5.0' 已存在!"  │
│  💡 提示: "请先删除现有 Tag 或使用新版本号"               │
│  🚫 退出执行                                            │
│                                                         │
│  解决方案:                                               │
│  1. 使用新版本号: v1.5.1                                │
│  2. 或删除旧 Tag:                                       │
│     git push --delete origin switch520/v1.5.0          │
│     git tag -d switch520/v1.5.0                        │
└─────────────────────────────────────────────────────────┘
```
